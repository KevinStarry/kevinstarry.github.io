Hugo项目的工程文件，public目录下是生成的静态文件，即网站的所有依赖文件。  

起初是想在CSDN上写写学习技术的一些文章，供自己以后参考。也是为了给别人一些参考意见。在简述上写一些心情日记，非技术类文章的。但是当我在CSDN上转载一篇终端代理的文章的时候，被CSDN审查为翻墙不予通过。  
奈何国内网络访问GitHub太慢，git clone一个项目实在是太慢了，所以有终端代理的需求就应运而生了。记录在自己电脑上也不方便随时查看，于是想着挂在互联网上，反正也不是什么私密的东西，纯属记录一些学习经验。  

之前Google Bloggers可以通过绑定域名的方式，大陆可以直接访问，但是Google Blogger用起来总感觉少了些什么，可能是没法把所有的都控制在手中吧，权限不够大。相比于Github的Page，可操作性少了些。  

在使用GithubPages的时候，考虑过Hexo，因为国人用的很多，但是不太喜欢node.js，总感觉电脑多一个东西不喜欢，而且本人十分畏惧js，头疼。虽然Hugo大部分是外国人在用，个人感觉，国外很多东西的教程都写的比较容易上手，可能是看的不多吧。  

日本人写的教程书尤其通俗易懂，可能是民族特性吧，这点挺好的，学起来不费劲。
